🌻 N-Lang: Manual de Ingeniería de Bits

N-Lang es un lenguaje de ejecución por inducción. Aquí no hay "saltos" condicionales lentos; hay flujo de acarreos y presión de registros.

1. Los Componentes del Circuito

N-Lang se basa en tres unidades fundamentales que se mapean directamente al silicio.

VAR (Acumulador de Umbral): No es una caja de datos, es un contador de 16 bits. Cuando el valor alcanza el umbral, el bit 15 se dispara (acarreo) y activa lo que esté debajo.

Sintaxis: VAR nombre (umbral, mensaje) += valor

CTRL (Puerta Lógica): Controla el flujo. El tipo 0 es un sumidero que resetea el estado, eliminando la "basura" en un solo ciclo de reloj.

CLASS (Topología): Define cómo se conectan los cables. Es una "macro" de hardware virtual.

2. El "GC de un Bit" (La Magia de las 10 líneas)

A diferencia de otros lenguajes que necesitan pausar todo para limpiar memoria, N-Lang usa la aritmética binaria:

Reposo: El registro está en 0x7FFF (0111...).

Activación: Recibe un acarreo y pasa a 0x8000 (1000...). El bit de signo se activa, se ejecuta la acción.

Auto-extinción: En el siguiente ciclo, el CTRL {0} simplemente lo devuelve a 0x7FFF.

Nota para Ingenieros: Esto es lo que hace que N-Lang sea más rápido que C. No hay gestión de memoria, solo oscilación de bits.

3. La "Sesentinidad" (Programación por Acarreo)

Mira cómo funciona un reloj en N-Lang. No necesitas if (segundos == 60). El hardware lo hace por ti:

Code snippet


VAR segundo (60) += 1       # Al llegar a 60, desborda...

VAR minuto (60)             # ...y este recibe un +1 automáticamente.

VAR hora (24)               # ...y así sucesivamente.

Es una reacción en cadena. Si envías 3600 al segundo, el acarreo viajará instantáneamente hasta la hora.

4. Lógica Branchless (Sin Saltos)

En N-Lang, el if y el else son simples inversiones de energía.

Code snippet


CLASS else(condición) = condición(!1)  # Invierte el bit de umbral

end = CTRL {0}                         # El aislante de flujo


if(A = B)

   print("Iguales")

end                                    # Corta la señal aquí

else(if)

   print("Diferentes")

5. Super-Variables (SIMD Natural)

Como cada VAR usa solo 16 bits, podemos empaquetar 4 en un registro de 64 bits (r13, rbx).

Portabilidad Universal: N-Lang usa la instrucción ADC (Add with Carry). Esto significa que corre igual de rápido en un Intel Core i9, un ARM de celular o un núcleo CUDA de GPU.

Escalabilidad: Si necesitas una variable de 256 bits, N-Lang simplemente encadena 4 registros AVX y deja que el acarreo fluya entre ellos.

6. Ejemplo Maestro: El Bucle For

Un for en N-Lang es un sistema de retroalimentación:

Code snippet


# El bucle se mantiene activo mientras 'in' sea menor a 'to'

CTRL bucle_for {return, in += (in < to) * each}

Si la condición (in < to) devuelve 0, el multiplicador apaga el flujo y el bucle muere. Cero saltos, máxima velocidad.

N-Lang: El Lenguaje Sin Saltos (Branchless) En otros lenguajes, el código "salta" de un lugar a otro, perdiendo tiempo y claridad. En N-Lang, el código fluye. Una condición no es una orden de saltar, es un umbral de energía. Si la condición no se cumple, el bit no tiene la "fuerza" para activar la siguiente instrucción. El resultado: código que se lee como un libro pero corre como la luz en un cable.
1. La Regla Rota: El Fin de los Saltos

En la programación estándar, el bajo nivel es difícil de leer porque está lleno de saltos (JMP, GOTO). El alto nivel es fácil de leer pero lento porque el CPU debe adivinar el camino.

N-Lang rompe la jerarquía:

    Alto Nivel: Se lee plano y elegante.

    Bajo Nivel: No hay saltos. Es una topología de transistores virtuales.

Comparativa de Ejecución:
Paradigma Tradicional (Saltos)	Paradigma N-Lang (Flujo)
1. Evaluar condición.	1. Cargar "presión" de bits en r15.
2. SALTAR a línea 5 si es falso.	2. El bit fluye a la siguiente instrucción.
3. Ejecutar instrucción A.	3. Si el umbral no se alcanza, el bit se bloquea.
4. SALTAR a línea 6.	4. Si el umbral se alcanza, la acción se induce.
5. Ejecutar instrucción B (Else).	Resultado: El pipeline del CPU nunca se detiene.
2. Los Componentes del Circuito
A. VAR (Acumulador de Umbral)

No es una variable de memoria; es un contador de 16 bits.

    Mecánica: Cuando sumas un valor, el acumulador se llena. Si cruza el umbral (Bit 15), dispara una señal hacia abajo.

    Sintaxis: VAR nombre (umbral, mensaje) += valor

B. CTRL (Puertas de Control)

Gestionan la dirección y el reset de la energía.

    CTRL {0} (El Aislante): Es el sumidero de energía. Corta el flujo de acarreos para separar procesos. Es el famoso GC (Garbage Collector) de un bit.

C. CLASS (Arquitectura de Cables)

Permite al usuario definir sus propios componentes lógicos (como if, else, for).

    Ejemplo: CLASS else(condición) = condición(!1) (Simplemente invierte la polaridad del bit).

3. El Misterio del GC de un Bit (Explicado para Humanos)

Los ingenieros se asustan porque N-Lang limpia la memoria en 1 ciclo de reloj usando aritmética, no software.

    Estado 01 (Inactivo): Registro en 0x7FFF. Listo para recibir carga.

    Estado 10 (Activado): Recibe un acarreo (+1), pasa a 0x8000. El bit de signo se enciende y dispara la acción.

    Reset Automático: El CTRL {0} simplemente "limpia" el acarreo, devolviendo el registro a 0x7FFF. Sin pausas, sin buscar basura.

4. Sesentinidad: El Dominó de Bits

Puedes crear sistemas complejos encadenando umbrales. En un reloj de N-Lang, el tiempo "cae" por gravedad aritmética:
Code snippet

VAR picofotograma (60) += entrada
VAR segundo (60)        # Se activa cuando picofotograma desborda
VAR minuto (60)         # Se activa cuando segundo desborda

5. Portabilidad Universal (ADC/SBC)

N-Lang es el lenguaje más portable del mundo porque solo usa las instrucciones más básicas del silicio: Add with Carry (ADC) y Subtract with Borrow (SBC).

    En x86: Usa registros de 64 bits.

    En ARM: Usa la densidad masiva de registros para tener cientos de variables vivas al mismo tiempo.

    En GPU/CUDA: Trata a los hilos de la tarjeta de video como una red de transistores en paralelo.

    Veredicto: Programar en N-Lang es como diseñar un procesador dentro de otro procesador. Es legible como el agua, pero corre a la velocidad del rayo.
